app/: 기능 서버 및 PWA 앱 (Node.js).  
app/data/: 사용자별 데이터 저장소 (username/~.json 형태).  
공통 요구사항:
디자인: 깔끔하고 고급스러운 디자인.  
뷰: 모바일과 데스크탑 뷰를 완벽히 분리하고, 모바일에서는 네이티브 앱 같은 UI 제공. 뷰(화면)마다 다른페이지로 설계. 
하드코딩을 최소화하고 유연하게 설계할 것.   
3. 기능 앱 서버 (app/) 상세 
기술 스택: Node.js,  Firebase Auth.  
인증 로직:
Firebase를 이용한 이메일/비밀번호 로그인 및 회원가입.  
비밀번호 초기화 기능 및 개인정보/이용약관 동의 필수 포함.  
테스트 계정: ID a@b.c / PW 3087  
데이터 및 설정:
모든 키(Key)와 URL은 루트의 keys.json에서 관리.  
API 요청은 /api/~ 형태로 구성.  
PWA: PWA 설정을 지원하되 캐싱은 제외하고, 앱 미설치 시 상단에 '설치 버튼'을 노출할 것. 푸시알람을 지원할것
4. 서비스 핵심 비즈니스 로직 (ser.txt 기반)
핵심 컨셉: 일정 자동 분배 및 재조정. 문제 풀이 패턴 분석보다는 '실행 강제'에 집중.  
AI 기능 (OpenAI 활용):
사용자의 남은 분량, 마감일, 가용 시간을 계산해 일정 분배.  
오프데이(휴무일) 제외 및 버퍼(가용 시간의 약 80%만 배정) 자동 포함.  
계획 미완료 시 '오늘 할 일 다음으로 미루기' 기능을 통한 자동 재분배.
